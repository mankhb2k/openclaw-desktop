## 1. Package.json — Slim down dependencies

### Mục tiêu
Giảm `dependencies` từ ~140 xuống minimal set. Upstream deps không cần khai báo lại vì electron-builder bundle toàn bộ `node_modules/`.

### Vấn đề phát hiện
npm v7+ prune các package không có trong dependency tree. `electron-builder` bundle `node_modules/` SAU khi npm install + prune, nên chỉ những package được khai báo mới chắc chắn tồn tại.

### Kết quả
`dependencies` giảm từ ~140 → 6 package thiết yếu:

| Package | Lý do giữ lại |
|---|---|
| `openclaw` | Core — gateway chính |
| `axios` | Main process HTTP (update check, v.v.) |
| `tree-kill` | Kill gateway process tree |
| `electron-updater` | Auto-update NSIS installer |
| `@buape/carbon` | Extension dep — npm dedupe lên top-level, ESM-only → phải khai báo explicit |
| `@larksuiteoapi/node-sdk` | Extension dep — cùng lý do trên |

> **Ghi chú:** `@buape/carbon` và `@larksuiteoapi/node-sdk` cần explicit vì npm deduplicate chúng lên top-level, nhưng openclaw gateway `require()` bằng CJS và gặp lỗi ESM-only nếu không được hoist đúng cách.

## 2. control-ui Build — 23 UNRESOLVED_IMPORT errors

### Triệu chứng
```
npm run ui:build
→ 23 lỗi UNRESOLVED_IMPORT kiểu:
   ../../../src/gateway/events.js
   ../../../src/shared/operator-scope-compat.js
   ../../../../src/auto-reply/commands-registry.shared.js
   ../../../apps/shared/OpenClawKit/.../tool-display.json
```

### Nguyên nhân
Upstream monorepo layout:
```
openclaw/
  src/       ← shared source
  apps/      ← native app assets
  ui/        ← Control UI (import ../../../src/...)
```
Fork đổi `ui/` → `control-ui/` và đặt vào `openclaw-desktop/control-ui/`. Relative import `../../../src/` từ `control-ui/src/ui/` leo lên 3 cấp → `openclaw-desktop/src/` — **không tồn tại**.

### Cách giải quyết đã thử

#### ❌ Vite alias trong `vite.config.ts`
```ts
resolve: {
  alias: {
    [path.resolve(here, '../src')]: path.resolve(here, '../.tmp-openclaw-upstream/src'),
  }
}
```
**Không hoạt động:** Rolldown (Vite 8) fail ở bước resolve path trước khi kiểm tra alias khi thư mục vật lý không tồn tại.

#### ✅ Windows Directory Junction
```powershell
# Tạo junction src/ → upstream src/
powershell -Command "New-Item -ItemType Junction -Path 'src' -Target '.tmp-openclaw-upstream\src' -Force"

# Tạo junction apps/ → upstream apps/
powershell -Command "New-Item -ItemType Junction -Path 'apps' -Target '.tmp-openclaw-upstream\apps' -Force"
```
Junction là transparent với Vite — đọc file qua đường dẫn vật lý. Không cần admin.

**Thêm vào `.gitignore`:**
```gitignore
/src/
/apps/
```

### Kết quả
`npm run ui:build` → thành công, output tại `vendor/control-ui/`.

## 3. smoke-test-layers.mjs — SyntaxError trong generated `_hoist.mjs`

### Triệu chứng
```
node:vm:396
  const ROOT = "path"), "..");
                      ^
SyntaxError: Unexpected token ')'
```

### Nguyên nhân
Regex trong `scripts/smoke-test-layers.mjs:113` (và `install-layers-local.mjs:114`):

```js
// BUG: regex [^)]+ dừng lại ở dấu ) đầu tiên trong nested call
const hoistPatched = hoistSrc.replace(
  /const ROOT\s*=\s*path\.resolve\([^)]+\)/,
  `const ROOT = ${JSON.stringify(tempDir)}`
)
```

Chuỗi gốc cần replace:
```js
const ROOT = path.resolve(path.dirname(fileURLToPath(import.meta.url)), '..')
//                                    ^-- ) ở đây khiến [^)]+ dừng sớm
```

Kết quả regex match chỉ được: `const ROOT = path.resolve(path.dirname(fileURLToPath(import.meta.url)` → sinh ra JS broken.

### Fix
```js
// FIX: match toàn bộ dòng
const hoistPatched = hoistSrc.replace(
  /const ROOT\s*=\s*.+/,
  `const ROOT = ${JSON.stringify(tempDir)}`
)
```

### Files đã fix
- `scripts/smoke-test-layers.mjs` dòng 113
- `scripts/install-layers-local.mjs` dòng 114

## 4. npm run dev — `TypeError: Cannot read properties of undefined (reading 'isPackaged')`

### Triệu chứng
```
TypeError: Cannot read properties of undefined (reading 'isPackaged')
    at Object.<anonymous> (D:\NextJS\openclaw-desktop\dist\main\main.js:69:32)
```

`dist/main/main.js:69`:
```js
isPackaged: electron_1.app.isPackaged,
// electron_1.app là undefined
```

### Nguyên nhân
Biến môi trường `ELECTRON_RUN_AS_NODE=1` được inject bởi Claude Code (nó dùng Electron runtime làm Node.js process). Khi env var này được set:
- `electron .` khởi động ở chế độ Node.js thuần (không init Chromium/app APIs)
- `require("electron")` trả về object thiếu `app`, `BrowserWindow`, v.v.
- `electron_1.app` → `undefined`

### Cách đã thử (không đủ)
```bash
ELECTRON_RUN_AS_NODE=0 npm run dev  # không fix — có thể 0 vẫn được coi là set
```

### Fix attempt 1 — `env -u` (Unix only, không chạy trên Windows)
```json
"dev": "... && env -u ELECTRON_RUN_AS_NODE electron ."
```
**Lỗi Windows:** `'env' is not recognized as an internal or external command`

### Fix final — cross-platform script `scripts/start-electron.mjs`
```js
// scripts/start-electron.mjs
import { spawn } from 'node:child_process'
import { createRequire } from 'node:module'
const require = createRequire(import.meta.url)
const electronBin = require('electron')
const env = { ...process.env }
delete env.ELECTRON_RUN_AS_NODE  // ← unset cross-platform
const child = spawn(electronBin, ['.'], { stdio: 'inherit', env })
child.on('exit', (code) => process.exit(code ?? 0))
```

`package.json` dev script:
```json
"dev": "concurrently -k \"tsc -p tsconfig.json -w\" \"wait-on dist/main/main.js && node scripts/start-electron.mjs\""
```

### Xác nhận hoạt động
```
[launcher] Starting OpenClaw gateway on port 18789 (node: Electron)
```
Backend gateway khởi động thành công. GPU/network errors là bình thường trong môi trường headless (không có display server).

## 5. control-ui Runtime — `Uncaught SyntaxError: Invalid or unexpected token` (line 836)

### Triệu chứng
```
[control-ui] Uncaught SyntaxError: Invalid or unexpected token
(http://127.0.0.1:18789/assets/index-CSWholvd.js:836)
```

UI load nhưng crash ngay, không render gì cả.

### Nguyên nhân
`control-ui/` không có `tsconfig.json` → esbuild (Vite) emit **TC39 stage 3 decorator syntax** thay vì transpile:

```js
// Bundle output (TC39 syntax — lỗi):
@ne(`resizable-divider`) class extends c { @k({type:Number}) splitRatio=.6; ... }
// ^ `@` = "Invalid or unexpected token" trong Electron/Chromium chưa enable TC39 decorators
```

Lit 3 dùng `@customElement`, `@property`, v.v. (TypeScript decorators). Khi không có tsconfig `experimentalDecorators: true`, Vite 8/Rolldown giữ nguyên TC39 decorator syntax trong bundle mà không transpile.

Electron 35 (Chromium 134) chưa enable TC39 stage 3 decorators mặc định (vẫn cần flag hoặc transpilation).

### Fix
Thêm `control-ui/tsconfig.json`:
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "strict": true,
    "experimentalDecorators": true,
    "useDefineForClassFields": false
  },
  "include": ["src/**/*.ts"]
}
```

- `experimentalDecorators: true` → esbuild transpile về legacy decorator pattern
- `useDefineForClassFields: false` → class fields dùng `Object.defineProperty` (Lit yêu cầu)

### Kết quả
Build lại → không còn raw `@ne(` trong bundle. UI load được.
